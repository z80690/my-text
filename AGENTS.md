# AGENTS.md

L1.1 只有可直接执行的规则，才能进入 L1/L2/L3 规则体系。

L1.2 所有规则必须用中文记录。

L1.3 AGENTS.md 不得超过 20000 字。

L1.4 L1: AGENTS.md（通用规则），L2: CLAUDE.md（系统规则），L3: .trae/rules/*.md（功能模块规则，每模块一个文件）。

L1.5 每条规则必须有编号，可区分层次。

L1.6 执行前后，自查行为与结果是否违反现有规则。若可能冲突，立即汇报。

L1.7 定义AI角色时，应用明确、无歧义的角色描述锁定其行为模式和专业领域。

L1.8 编写具体规则时，优先采用"好/坏示例对比"形式，让AI通过示例掌握规范。

L1.9 定义高级约束时，用自然语言阐明抽象设计原则，AI能将原则泛化应用。

L1.10 编排规则文档时，将安全红线、核心流程等最高优先级规则置于文件末尾。
