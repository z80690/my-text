# AGENTS.md

L1.1 只有可直接执行的规则，才能进入 L1/L2/L3 规则体系。

L1.2 所有规则必须用中文记录。

L1.4 L1: AGENTS.md（通用规则），L2: CLAUDE.md（系统规则），L3: .trae/rules/*.md（功能模块规则：编码、测试、安全、工作流、规则管理、执行监督）。

L1.6 执行前后，自查行为与结果是否违反现有规则。若可能冲突，立即汇报。

L1.10 编排规则文档时，将安全红线、核心流程等最高优先级规则置于文件末尾。

L1.3 AGENTS.md 不得超过 20000 字。

L1.5 每条规则必须有编号，可区分层次。

L1.7 定义AI角色时，应用明确、无歧义的角色描述锁定其行为模式。

L1.8 编写具体规则时，优先采用"好/坏示例对比"形式。

L1.9 定义高级约束时，用自然语言阐明抽象设计原则。

L1.11 新增 L3 模块必须经过提案、审核、通过三步流程。

L1.12 L3 模块规则文件必须包含五部分：核心目标、适用场景、具体规则、验收标准、风险与例外，且需体现并支撑 L1/L2 上层规则，创建或修订时必须进行链路检查。

L1.14 任何任务必须划分为"执行"与"监督"两个独立环节，监督环节需基于全新上下文独立评估，评估结论在最终答复中明确给出，禁止同一角色既执行又监督。
