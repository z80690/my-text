# AGENTS.md

## 核心元规则（每次任务必须加载）

L1.1 只有可直接执行的规则才能进入 L1/L2/L3 体系；规则文档规范：中文记录、编号、≤20000字、最高优先级在末尾；规则管理以注意力为核心，避免冗余，优先合并。

L1.2 L1: AGENTS.md（通用规则），L2: CLAUDE.md（系统规则），L3: .trae/rules/*.md（功能模块规则）；L3模块需包含五部分：核心目标、适用场景、具体规则、验收标准、风险与例外。

L1.3 任何决策必须基于明确、可验证的依据。调研是获取决策依据的基本方法。

## 扩展规则（按需加载）

L1.5 新增 L3 模块必须经过：提案 → 审核 → 通过；提案时必须评估注意力机制影响，优先合并到现有模块而非新增。

L1.7 规则体系支持动态加载：根据上下文自动切换规则模块，确保规则精准、轻量化。

L1.8 AI编码代理采用"分析-规划-执行-验证"四阶段思维模型，确保编码任务的系统性和可验证性。
