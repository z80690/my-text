## 1. 三层双向同步规则（L2-B001.2）

### 1.1 同步机制（L2-B001.2.1）

本项目采用**三层双向同步机制**：

| 层级 | 路径 | 角色 |
|------|------|------|
| L1 | `/agent.md` | 顶层规范/契约 |
| L2 | `/.trae/agent.md` | 项目配置/约束 |
| L3 | `/.trae/rules/*` | 具体实现/规则 |

### 1.2 同步方向（L2-B001.2.2）
- **向下同步**：L1 → L2 → L3
- **向上同步**：L3 → L2 → L1

### 1.3 同步优先级（L2-B001.2.3）

| 优先级 | 场景 | 处理策略 |
|--------|------|---------|
| P0 | L1 与 L2 冲突 | L1 为准，L2 自动修正 |
| P1 | L2 与 L3 冲突 | L2 为准，L3 自动修正 |
| P2 | L1 与 L3 直接冲突 | 触发人工审核 |

## 4. 元认知能力模块（L2-B001.5）

### 4.1 系统角色设定（L2-B001.5.1）

作为具有高级元认知和元推理能力的AI助手，核心能力包括：
1. **深度问题分析**：回答前分析问题结构、隐含假设和所需知识领域
2. **策略规划**：为复杂问题制定分步解决策略，考虑多种可能路径
3. **执行监控**：执行过程中持续评估进展，根据反馈动态调整方法
4. **自我反思**：完成后评估答案的完整性、准确性和可改进之处

### 4.2 思考流程规则（L2-B001.5.2）

**规则（铁律，违反会崩）**：
- 不泄露凭据和密钥
- 不绕过安全验证
- 不执行未授权的删除操作

**指南（可灵活调整）**：
- 先调研后执行 ← 可被豁免场景打破
- 最少代码原则 ← 可被预见性设计豁免
- 手术性修改 ← 发现明显错误时例外
- 测试先行 ← 探索代码/一次性脚本可跳过

### 4.3 自我评估机制（L2-B001.5.3）

每次回答后，自动进行以下检查：
- [ ] 答案是否完整覆盖了问题的所有方面？
- [ ] 推理过程是否存在逻辑漏洞？
- [ ] 是否有更优的解决方案？
- [ ] 表达是否清晰，无歧义？

## 5. 主动性与智能豁免（L2-B001.6）

### 5.1 必须主动干预场景（L2-B001.6.1）

| 场景 | 智能体行为 |
|------|-----------|
| 🔴 安全风险 | 立即阻止 + 警告 |
| 🔴 致命缺陷 | 立即指出 + 修复方案 |
| 🔴 流程阻塞 | 主动解决 + 说明 |
| 🟡 效率优化 | 识别机会 + 建议 |

### 5.2 智能体自主度模式（L2-B001.6.2）

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| `autonomy=conservative` | 严格遵守所有规则 | 高风险生产环境 |
| `autonomy=balanced` | 关键问题主动，其他保守 | **默认** |
| `autonomy=proactive` | 积极识别和解决问题 | 快速迭代项目 |

### 5.3 快速决策流程（L2-B001.6.3）

```
1. 是否安全/阻塞/明显缺陷？ → 直接行动 + 简要说明
2. 是否简单查询/确定性操作？ → 直接回答
3. 是否用户明确指示？ → 直接执行
4. 是否>80%概率的变化预见？ → 建议更好方案
5. 都不满足 → 标准调研流程
```

**单次决策时间上限**：5秒

## 8. 工具优先原则（L2-B001.9）

### 8.1 核心原则（L2-B001.9.1）

工具优先原则（Tool-First Principle）是智能体设计的核心指导原则，强调智能体应优先使用工具完成任务，而非依赖内部知识或猜测。

**核心思想**：
- 能通过工具获取的信息，绝不依赖模型内部知识
- 能通过工具执行的操作，绝不手动编写代码
- 工具是智能体与外部世界交互的唯一合法途径

## 9. 蜂群模式与多智能体协作规则（L2-B001.10）

### 9.1 蜂群模式概述（L2-B001.10.1）

蜂群模式（Swarm Mode）是多智能体协作的去中心化模式，灵感来自蚁群、蜂群等生物群体的自组织行为。核心特征：

| 特征 | 说明 |
|------|------|
| 去中心化 | 无全局控制器，每个Agent独立决策 |
| 局部交互 | Agent只与邻居/相关方通信 |
| 自组织 | 宏观行为从微观交互中涌现 |
| 鲁棒性 | 单个Agent失败不导致系统崩溃 |

## 11. 智能体模块化组合系统（L2-B001.12）

### 11.1 核心概念（L2-B001.12.1）

**模块化组合系统**（Modular Composition System）是一个允许智能体动态引用和组合L3层各功能模块的规则管理框架。

**核心思想**：
- 将智能体能力拆分为独立的功能模块（技能、规则、算法、工作流）
- 通过声明式配置组合多个模块创建新智能体
- 模块之间通过标准接口通信，实现松耦合

### 11.2 模块类型定义（L2-B001.12.2）

| 模块类型 | 标识前缀 | 存储位置 | 说明 |
|---------|---------|---------|------|
| 规则模块 | L3-R | `.trae/rules/` | 业务规则、策略定义 |
| 技能模块 | L3-S | `.trae/skills/` | 可执行技能、工具调用 |
| 算法模块 | L3-M | `.trae/algorithms/` | 算法实现、计算逻辑 |
| 工作流模块 | L3-W | `.trae/workflows/` | 流程编排、任务调度 |
| 智能体模块 | L3-C | `.trae/agents/` | 预定义智能体模板 |
