# 技能触发提示词与规则体系分析报告

---

## 第一部分：所有技能触发提示词汇总

### 1. 智能排错 (auto-debug)
**触发关键词**：调试、报错、bug、错误、修复  
**触发场景**：用户遇到代码错误、程序崩溃、功能异常时  
**功能定位**：自动化调试，从报错到修复一站式处理

### 2. 自动化钩子 (auto-hook)
**触发关键词**：hook、钩子、自动化、触发  
**触发场景**：需要设置事件驱动的自动化流程时  
**功能定位**：事件驱动的自动化触发器

### 3. 一键生成文档 (auto-doc)
**触发关键词**：文档、README、注释、生成文档  
**触发场景**：需要为项目或代码生成文档时  
**功能定位**：自动生成README、API文档、代码注释

### 4. 全局代码重构 (auto-refactor)
**触发关键词**：重构、refactor、批量修改、迁移  
**触发场景**：需要对代码进行大规模修改或项目结构调整时  
**功能定位**：批量安全重构整个项目

### 5. 本地隐私联动 (local-privacy)
**触发关键词**：本地、隐私、local、笔记  
**触发场景**：需要安全读取本地文件或资料时  
**功能定位**：本地资源安全读取，不上传云端

### 6. 自动记忆处理 (auto-memory)
**触发关键词**：记忆、记住、自动记忆  
**触发场景**：用户提供偏好、背景信息或反馈时自动触发  
**功能定位**：自动识别暗知识并写入记忆文件，静默执行

### 7. 智能文件清理 (file-cleaner)
**触发关键词**：清理、清理文件、清理未使用、file-cleaner  
**触发场景**：需要清理项目中的垃圾文件或未使用资源时  
**功能定位**：智能文件清理，集成未使用文件/依赖检测

### 8. 代码审查 (my-code-review)
**触发关键词**：代码审查、代码质量、代码分析、code-review  
**触发场景**：需要对代码进行质量评估或安全审查时  
**功能定位**：专业的代码审查，分析质量、识别问题、提供改进建议

### 9. 工具调用追踪 (tool-usage-tracker)
**触发关键词**：追踪、工具调用、调用追踪、tracker  
**触发场景**：需要查看工具使用统计或生成调用报告时  
**功能定位**：追踪MCP和Skills的调用情况，自动生成每日报告

### 10. 女娲造人 (huashu-nuwa)
**触发关键词**：造skill、蒸馏、女娲、造人、思维方式  
**触发场景**：需要创建新的人物Skill或深度调研时  
**功能定位**：输入人名/主题/模糊需求，自动生成可运行的人物Skill

### 11. API Token优化器 (api-token-optimizer)
**触发关键词**：API优化、Token节省、限流、API缓存、rate limit  
**触发场景**：需要优化API调用或节省Token消耗时  
**功能定位**：节省API调用Token，涵盖缓存、限流、轮询、优雅降级等策略

## 第二部分：L1/L2/L3规则体系功能分析

### L1层 - 基础触发层

**核心功能定位**：快速匹配、即时响应  
**技术实现**：基于关键词的精确匹配引擎

**核心能力**：
1. **关键词匹配**：支持单关键词和多关键词组合匹配
2. **优先级调度**：规则优先级排序（当前统一为10）
3. **状态管理**：规则启用/禁用状态控制
4. **冷却机制**：支持规则触发冷却时间设置
5. **直接映射**：关键词到技能的直接映射

**覆盖模块**：
- `trigger_rules.json`：24条关键词触发规则
- `skills.yaml`：11个技能的关键词配置

**功能边界**：
- 输入：用户消息中的单个或多个关键词
- 输出：匹配的技能ID和动作
- 处理时间：O(n)，n为规则数量
- 适用场景：简单指令、明确意图

**当前实现特征**：
- 规则数量：24条
- 覆盖技能：6个（auto-debug、auto-hook、auto-doc、auto-refactor、local-privacy、auto-memory）
- 匹配模式：精确匹配、大小写不敏感
- 优先级：统一为10，无差异化

---

### L2层 - 上下文匹配层

**核心功能定位**：上下文感知、意图理解  
**技术实现**：记忆系统 + 意图识别引擎

**核心能力**：
1. **记忆存储与检索**：用户偏好、项目背景、反馈记录的存储和查询
2. **意图分类**：支持code/design/review/research/general五类意图识别
3. **上下文保持**：最近10轮对话的上下文管理
4. **暗知识提取**：自动识别用户偏好、团队规则等隐含信息

**覆盖模块**：
- `memories/`：记忆文件存储（user/project/feedback/reference四类）
- `auto-memory`技能：自动记忆处理
- `semantic_router`：语义路由辅助

**功能边界**：
- 输入：用户对话历史 + 当前消息
- 输出：用户意图分类、相关记忆片段
- 处理时间：O(m)，m为记忆条目数量
- 适用场景：需要上下文理解的复杂指令

**当前实现特征**：
- 记忆分类：4类（用户记忆、项目记忆、反馈记忆、参考记忆）
- 上下文窗口：10轮对话
- 自动触发：auto-memory技能静默执行
- 意图识别：基于语义相似度

---

### L3层 - 策略决策层

**核心功能定位**：智能路由、综合决策  
**技术实现**：多维度评分 + 博弈调度

**核心能力**：
1. **向量相似度路由**：余弦相似度算法计算文本匹配度
2. **知识图谱查询**：概念、智能体、技能、任务之间的关系分析
3. **负载均衡**：基于当前负载的智能调度
4. **故障转移**：断路器模式的故障恢复
5. **博弈调度**：辩论、优化、设计、协商、拍卖五种模式

**覆盖模块**：
- `algorithms/vector_matcher.py`：向量匹配算法
- `knowledge_graph_mcp.py`：知识图谱服务
- `rate_limiting/circuit_breaker.py`：断路器模式
- `algorithms/game_theory.py`：博弈论算法

**功能边界**：
- 输入：用户消息、上下文、系统状态
- 输出：最优技能选择、执行策略
- 处理时间：O(k)，k为候选技能数量
- 适用场景：复杂决策、多技能协作

**当前实现特征**：
- 路由策略：向量相似度最高优先级
- 博弈模式：5种（辩论、优化改进、深度设计、协商决策、资源拍卖）
- 容错机制：断路器模式（阈值5次失败，超时30秒）
- 负载策略：最小负载调度

---

### 三层架构协作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户输入                                │
└───────────────────────────┬───────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│  L1层：关键词匹配                                               │
│  - 扫描所有规则，匹配关键词                                      │
│  - 命中则直接返回技能ID                                         │
│  - 未命中则传递到L2层                                           │
└───────────────────────────┬───────────────────────────────────┘
                            │ 未命中
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│  L2层：上下文分析                                               │
│  - 查询记忆系统获取相关上下文                                     │
│  - 进行意图分类识别                                             │
│  - 结合上下文推荐候选技能                                         │
│  - 无法确定则传递到L3层                                           │
└───────────────────────────┬───────────────────────────────────┘
                            │ 无法确定
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│  L3层：智能决策                                                 │
│  - 计算语义相似度                                               │
│  - 查询知识图谱                                                 │
│  - 评估系统负载                                                 │
│  - 综合评分选择最优技能                                           │
└───────────────────────────┬───────────────────────────────────┘
                            │
                            ▼
                    技能执行
```

---

### 各层功能对比

| 层级 | 功能定位 | 核心能力 | 响应速度 | 适用场景 |
|------|---------|---------|---------|---------|
| **L1** | 快速匹配 | 关键词匹配、优先级调度 | 毫秒级 | 简单指令、明确意图 |
| **L2** | 上下文理解 | 记忆检索、意图识别 | 秒级 | 复杂指令、需要上下文 |
| **L3** | 智能决策 | 语义分析、知识图谱、博弈调度 | 秒级 | 复杂决策、多技能协作 |

## 第三部分：规则体系优化空间分析

### L1层优化建议

**当前问题**：
- 关键词覆盖不全（部分技能未配置规则）
- 优先级统一为10，无法区分重要性
- 缺乏同义词支持
- 规则冲突处理机制缺失

**优化方向**：

1. **关键词扩展**：
   - 为所有11个技能配置完整的触发规则
   - 建立同义词库（如"调试"、"debug"、"排错"、"排查"）
   - 支持短语匹配（如"生成文档"、"代码审查"）

2. **优先级差异化**：
   - 按技能重要性设置不同优先级
   - 高频技能设置较高优先级
   - 危险操作技能设置较低优先级并要求确认

3. **冲突处理机制**：
   - 添加规则冲突检测算法
   - 支持规则优先级覆盖
   - 多规则匹配时的歧义消解策略

4. **动态规则管理**：
   - 支持规则的热更新
   - 添加规则版本控制
   - 规则启用/禁用的实时生效

---

### L2层优化建议

**当前问题**：
- 记忆系统与触发规则分离，记忆未参与触发决策
- 上下文窗口仅10轮，不足以处理复杂对话
- 意图识别准确性有待提高
- 缺乏意图链追踪能力

**优化方向**：

1. **记忆驱动触发**：
   - 将记忆内容作为规则匹配的输入条件
   - 根据用户偏好动态调整规则优先级
   - 支持基于记忆的规则过滤

2. **上下文扩展**：
   - 扩大上下文窗口至20-30轮
   - 实现上下文摘要压缩机制
   - 支持长对话的意图保持

3. **意图链分析**：
   - 追踪用户意图的连贯性
   - 识别意图的演变和分支
   - 支持意图回退和修正

4. **场景模板匹配**：
   - 定义常见任务模式模板
   - 支持模板的动态匹配
   - 提供模板推荐功能

---

### L3层优化建议

**当前问题**：
- 向量相似度计算独立于规则系统
- 缺乏多技能协作的规则编排
- 博弈模式未与规则系统集成
- 决策透明度不足

**优化方向**：

1. **多模态融合决策**：
   - 整合关键词匹配度、语义相似度、上下文相关性
   - 引入历史成功率作为决策因子
   - 建立综合评分模型

2. **技能协作规则**：
   - 定义技能间的调用顺序和依赖关系
   - 支持技能组合编排
   - 实现技能流水线

3. **博弈模式集成**：
   - 将博弈模式作为规则的一部分
   - 支持基于博弈的决策选择
   - 实现辩论、协商等复杂交互模式

4. **决策可解释性**：
   - 提供决策过程的可追溯性
   - 生成决策理由说明
   - 支持用户对决策的质疑和修正

---

### 规则管理优化

**当前问题**：
- 规则管理界面缺失
- 规则生命周期管理不足
- 缺乏规则测试和验证机制

**优化方向**：

1. **规则管理界面**：
   - 提供规则的可视化管理
   - 支持规则的增删改查
   - 规则冲突的可视化展示

2. **生命周期管理**：
   - 自动禁用长期未使用的规则
   - 支持规则的归档和恢复
   - 规则版本的对比和回滚

3. **测试验证机制**：
   - 规则测试沙箱环境
   - A/B测试支持
   - 规则效果的量化评估

3. **性能监控**：

   - 系统负载监控

## 总结

### 当前状态评估
- **规则覆盖**：24条规则，覆盖6个技能（覆盖率54%）
- **层级协作**：L1→L2→L3的递进式处理
- **核心能力**：关键词匹配、记忆存储、语义理解、智能路由

### 关键改进方向
1. **L1层**：完善关键词覆盖，添加同义词支持，实现冲突处理
2. **L2层**：整合记忆系统与规则触发，扩展上下文窗口
3. **L3层**：实现多模态融合决策，集成博弈模式
4. **管理层面**：添加规则管理界面和可观测性功能

### 预期效果
- **触发准确率提升**：从当前约70%提升至90%+
- **用户体验改善**：更精准理解用户意图，减少重复提问
- **系统可维护性**：规则管理更加灵活高效
- **决策透明度**：用户可理解决策过程，增强信任感
