# Git工作流

## 核心功能
定义分支命名规范，区分main保护分支与功能分支
制定提交信息标准，统一feat/fix/docs等类型格式
建立Code Review流程，确保代码质量与规范符合要求
设置合并门禁规则，必须通过所有检查才能合并代码

## 执行步骤
定义分支策略，区分main、feat、fix、docs等分支类型
制定提交信息规范，要求类型+描述的标准格式
建立PR/MR审查流程，明确审查要点与通过条件
配置合并门禁，设置自动检查与人工审查要求

## 约束规则
main分支必须设置保护，禁止直接推送
提交信息必须符合规范，否则拒绝提交
Code Review必须至少两名开发者批准
合并前必须通过所有自动化检查

## 关联模块
编码规范、质量保障、测试指南、规则引擎、规则管理
