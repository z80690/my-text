# 规则建议机制 (L3)

**版本**: v1.0 | **更新日期**: 2026-05-21 | **规则状态**: ✅ 生效中
**触发条件**: 规则效果不佳、失败模式重复、规则冲突频繁
**L1支撑**: 调研与知识管理 [L1-12.3]
**L2支撑**: 规则变更规范、调研规范

---

## 一、触发场景

| 触发场景 | 建议类型 | 优先级 | 数据阈值 |
|---------|---------|-------|---------|
| 某规则频繁触发但效果不好 | 修改规则建议 | P1 | 5次以上触发，且多次被修正 |
| 新技术出现 | 添加技术规范建议 | P2 | 持续学习行业新技术 |
| 任务失败模式重复 | 添加检查清单建议 | P1 | 3次以上相同失败模式 |
| 规则冲突频繁发生 | 优化规则优先级建议 | P2 | 3次以上相同冲突 |
| 用户多次修正同一问题 | 新增规则建议 | P1 | 3次以上用户修正 |

---

## 二、建议流程

```
规则引擎收集执行数据
    ↓
分析模式识别优化机会
    ↓
生成规则建议草案
    ↓
提交用户审批（L1变更需用户明确批准）
    ↓
用户批准？→ 是 → 同步更新L1/L2/L3
    ↓ 否 → 归档建议
    ↓
更新知识图谱
```

---

## 三、建议草案格式

### 3.1 建议模板
```markdown
## 规则建议 [类型：修改/新增/删除]

### 触发原因
- 触发场景：[描述触发条件]
- 数据证据：[列出支持建议的数据]

### 建议内容
- 原规则：[当前规则内容]
- 建议修改：[具体修改建议]

### 预期影响
- 正面影响：[列出预期的改进]
- 潜在风险：[列出可能的风险]

### 推荐优先级
P1/P2/P3

---
提交日期：YYYY-MM-DD
```

### 3.2 建议示例
```markdown
## 规则建议 [类型：新增]

### 触发原因
- 触发场景：用户多次修正同一问题 - 密码明文存储
- 数据证据：3次任务中发现明文存储密码，均被用户指出修正

### 建议内容
- 原规则：无明确密码存储规范
- 建议修改：新增"密码必须使用bcrypt加密"规则

### 预期影响
- 正面影响：大幅提升安全性，避免安全隐患
- 潜在风险：无

### 推荐优先级
P1
```

## 四、建议审批机制

| 建议类型 | 审批要求 | 变更范围 |
|---------|---------|---------|
| 修改L1规则 | 用户明确批准 | L1/L2/L3同步 |
| 新增L1规则 | 用户明确批准 | L1/L2/L3同步 |
| 修改L2规则 | 用户确认 | L2/L3同步 |
| 新增L2规则 | 用户确认 | L2/L3同步 |
| 修改L3规则 | 系统自动（记录版本） | L3 |
| 新增L3规则 | 系统自动（记录版本） | L3 |

---

## 五、建议归档与追踪

| 状态 | 说明 | 后续动作 |
|------|------|---------|
| 待审批 | 提交用户未决策 | 等待用户回复 |
| 已批准 | 用户同意执行 | 执行变更，更新知识图谱 |
| 已拒绝 | 用户不同意执行 | 归档，记录拒绝原因 |
| 待重审 | 用户要求修改后重提 | 修改建议草案重新提交 |
