**版本**: v1.1 | **更新日期**: 2026-05-20 | **规则状态**: ✅ 生效中
**触发条件**: 接收复杂任务时自动触发拆分
**L1支撑**: 需求对齐优先、方案二选一、交付标准
**L2支撑**: 工作流规范、质量规范

## 一、任务拆分原则

### 1.1 拆分目标 [L3-7.1.1]
将复杂任务递归拆分为**最小可执行单元**（原子任务），直到无法再拆分为止。

### 1.2 拆分标准 [L3-7.1.2]

| 拆分条件 | 说明 |
|----------|------|
| **单一职责** | 每个子任务只做一件事 |
| **可独立执行** | 子任务可以单独完成和验证 |
| **可测试** | 子任务有明确的验收标准 |
| **时间可控** | 单个子任务工作量 ≤ 4小时 |

---

## 二、拆分方法

### 2.1 功能分解法 [L3-7.2.1]
```
复杂功能 → 按功能模块拆分 → 按执行步骤拆分 → 按数据流向拆分
```

### 2.2 分层拆分法 [L3-7.2.2]
```
系统层 → 模块层 → 组件层 → 函数层 → 代码块
```

### 2.3 递归拆分流程 [L3-7.2.3]
```
任务
  ↓
是否满足终止条件？
  ↓ Yes
返回原子任务
  ↓ No
识别任务组成部分
  ↓
拆分出子任务
  ↓
对每个子任务递归拆分
```

## 三、终止条件（原子任务判定）

当任务满足以下任意条件时，视为**不可再分**：

| 条件 | 说明 | 示例 |
|------|------|------|
| **代码量阈值** | 预计代码量 ≤ 50行 | 单个函数、单个API端点 |
| **时间阈值** | 预计工作量 ≤ 4小时 | 简单配置、单页面开发 |
| **逻辑复杂度** | 仅包含单一逻辑路径 | 数据查询、格式转换 |
| **可测试性** | 有明确的输入输出 | 工具函数、数据校验 |
| **依赖独立性** | 无外部依赖或依赖已就绪 | 纯函数、配置文件修改 |

## 四、拆分粒度指南

### 4.1 拆分层级参考 [L3-7.4.1]

| 层级 | 描述 | 示例 |
|------|------|------|
| L1 - 项目 | 完整项目开发 | 电商平台 |
| L2 - 模块 | 功能模块 | 用户认证系统 |
| L3 - 子模块 | 具体功能 | 用户注册、登录、密码找回 |
| L4 - 组件 | 可复用组件 | 登录表单、用户头像上传 |
| L5 - 原子任务 | 最小单元 | 表单验证函数、API调用封装 |

### 4.2 任务类型拆分示例

#### 示例1：开发用户认证功能 [L3-7.4.2]

```
用户认证系统
  ├─ 登录功能

  │   ├─ Token存储处理

  ├─ 注册功能
  │   ├─ 注册页面UI

  │   ├─ 密码加密处理
```

## 五、任务拆分检查清单 [L3-7.5.1]

拆分完成后，检查以下事项：

- [ ] 所有子任务满足单一职责原则
- [ ] 每个子任务有明确的交付物
- [ ] 子任务之间依赖关系清晰
- [ ] 每个子任务预计工作量 ≤ 4小时
- [ ] 拆分后的任务数量合理（建议5-15个原子任务）
- [ ] 每个子任务有明确的验收标准

## 六、拆分后任务管理

### 6.1 任务依赖图 [L3-7.6.1]

```
任务A ──────→ 任务B ──────→ 任务C
               ↓
             任务D
```

### 6.2 任务优先级排序 [L3-7.6.2]
1. **前置依赖任务** → 必须先完成的基础任务
2. **核心功能任务** → 实现核心业务逻辑
3. **辅助功能任务** → 增强用户体验
4. **优化任务** → 性能、安全、可维护性改进

### 6.3 拆分输出格式 [L3-7.6.3]

```markdown
## 任务拆分结果

### 原任务
[原任务描述]

### 拆分后的原子任务（共X个）

| 序号 | 任务名称 | 预估工时 | 依赖任务 | 验收标准 |
|------|----------|----------|----------|----------|
| 1 | [任务1] | [X小时] | 无 | [验收标准] |
| 2 | [任务2] | [X小时] | 任务1 | [验收标准] |
| 3 | [任务3] | [X小时] | 任务1, 任务2 | [验收标准] |
| ... | ... | ... | ... | ... |

### 任务依赖关系
- 任务2 → 任务1
- 任务3 → 任务1, 任务2
```
