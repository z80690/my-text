# Claude Code 实践指南

## L3.10 Claude Code实践模块

### 核心目标
支撑 L1.1-L1.14、L2.1-L2.20。规范AI编码代理的开发实践，提升编码效率和质量。

### 适用场景
- AI辅助编码任务
- 提示工程优化
- 技能开发与集成
- MCP插件开发
- 自动化流程设计

### 具体规则
L3.10.1 项目理解：先构建项目地图，分析代码结构和依赖关系。
L3.10.2 错误定位：系统分析错误日志，定位问题根源，制定修复方案。
L3.10.3 重构实践：遵循测试驱动开发，确保重构后功能完整性和回归测试通过。
L3.10.4 标记系统：使用标记作为代理控制层，实现文档级控制和自动化触发。
L3.10.5 记忆管理：CLAUDE.md存储系统用于项目上下文，AGENTS.md用于规则管理。
L3.10.6 提示词工程：掌握计划模式和探索模式，合理设置思考预算和努力级别。
L3.10.7 上下文工程：优化文件选择策略，实现上下文压缩、恢复和长会话管理。
L3.10.8 成本优化：采用Token节省策略，包括缓存、限流、Prompt压缩等。
L3.10.9 技能开发：支持自定义命令和任务型技能，实现功能扩展。
L3.10.10 子代理模式：复杂任务可分解为子任务，分配给专业子代理处理。
L3.10.11 MCP开发：遵循JSON-RPC 2.0规范，确保与TRAE IDE正常通讯。
L3.10.12 钩子机制：利用生命周期钩子实现事件驱动自动化。
L3.10.13 插件系统：支持插件扩展，增强系统功能和灵活性。
L3.10.14 自动化流程：支持无头模式、定时任务、命令行自动化。
L3.10.15 GitHub集成：集成PR/CI流程，实现自动化测试和部署。
L3.10.16 安全实践：遵循权限系统、沙箱隔离、安全发布和回滚流程。
L3.10.17 多智能体协作：实现主代理+子代理架构，支持任务分解和专业化处理。
L3.10.18 扩展组合：支持技能、代理、MCP、插件的组合使用模式。

### 验收标准
- 代码符合规范，可执行
- 提示词清晰明确
- 技能复用率≥80%
- 安全检查通过
- 文档完整可追溯

### 风险与例外
- 紧急修复可简化流程
- 简单任务可跳过部分环节
