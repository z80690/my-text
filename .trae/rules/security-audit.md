## 一、密钥管理

### 1.1 零容忍硬编码 [L3-4.1.1]
**绝对禁止**以下行为：
```python
# ❌ 绝对禁止
API_KEY = "sk-live_123456789abcdef"
DB_PASSWORD = "my-secret-password"
```

### 1.2 正确做法 [L3-4.1.2]
```python
# ✅ 正确：使用环境变量
import os
API_KEY = os.getenv("OPENAI_API_KEY")

# ✅ 正确：使用配置文件
from pydantic_settings import BaseSettings
class Settings(BaseSettings):
    api_key: str
settings = Settings()  # 从 .env 加载
```

### 1.3 示例代码中的占位符 [L3-4.1.3]
```python
# ✅ 使用占位符
API_KEY = "YOUR_API_KEY_HERE"
# 或
API_KEY = "sk-..."  # 省略实际值
```

## 二、依赖安全

### 2.1 安全检查清单 [L3-4.2.1]
- [ ] 检查依赖的安全公告：`npm audit`, `pip-audit`, `go vulncheck`
- [ ] 避免使用已知有漏洞的版本
- [ ] 定期更新依赖包
- [ ] 使用 lockfile (package-lock.json, Pipfile.lock)

### 2.2 风险依赖处理 [L3-4.2.2]
发现高危漏洞时：
1. 立即通知用户
2. 提供升级路径
3. 必要时提供替代方案

## 三、操作安全

### 3.1 风险操作范围 [L3-4.3.1]
| 操作类型 | 风险等级 | 是否需要审批 |
|----------|----------|--------------|
| 删除文件/目录 | 高 | ✅ 需要 |
| 覆盖现有文件 | 高 | ✅ 需要 |
| 大规模重写 | 中 | ✅ 需要 |
| 安装新依赖 | 中 | ⚠️ 建议 |

### 3.2 操作前检查 [L3-4.3.2]
执行风险操作前，必须：
1. 列出影响范围
2. 提供操作预览
3. 获得用户明确确认
