def load_skills():
    skills = []

    return sort_by_priority(skills)

## 2. Curator系统 - 技能自动优化（L3-R019）

### 2.1 评估指标

| 指标 | 说明 | 权重 |
|------|------|------|
| 成功率 | 技能成功完成任务的比例 | 40% |
| 使用频率 | 技能被调用的次数 | 30% |
| 用户反馈 | 用户对技能的评价 | 20% |
| 性能 | 执行时间、资源消耗 | 10% |

### 2.2 评分规则

```
总分 = (成功率 × 0.4) + (使用频率 × 0.3) + (用户反馈 × 0.2) + (性能 × 0.1)
```

| 评分范围 | 动作 |
|----------|------|
| ≥ 90 分 | **保留 + 推荐** |
| 70-89 分 | **保留** |
| 50-69 分 | **优化建议** |
| < 50 分 | **标记淘汰** |

### 2.3 配置选项

```yaml
curator:
  enabled: true
  auto_optimize: true
  auto_prune: false  # 谨慎启用
```

## 3. 自主进化机制 v2.0（L3-R014）

### 3.1 进化周期

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 触发周期 | 48小时 | 自动触发调研的时间间隔 |
| 提案存储 | `.trae/memories/project/evolution_proposals/` | 历史提案记录位置 |

### 3.2 递归调研机制

**递归调研** = 对每个L3子模块进行深度调研，然后对子模块内部的每个功能点再进行深度调研，形成树状调研结构。

### 3.3 多信源调研

| 信源 | 说明 | 调研要求 |
|------|------|----------|
| **A. TRAE/Cline官方** | TRAE官方技能系统 | ✅ 必须调研 |
| **B. GitHub Copilot** | Copilot官方技能和扩展 | ✅ 必须调研 |
| **C. Claude Code** | Claude Code官方技能 | ✅ 必须调研 |
| **D. 其他AI助手** | Cursor、Tabnine等第三方 | ✅ 必须调研 |

**⚠️ 强制要求：必须对所有信源A/B/C/D进行完整调研**

### 3.4 博弈评估流程

```
子功能调研
    ↓
【生成阶段】生成3-5个候选解决方案
    ↓
【博弈阶段】每个方案经过：
    ├── 拥护者辩护
    ├── 挑剔者质疑
    └── 成本分析
    ↓
【裁决阶段】用户选择最优方案
    ↓
【执行阶段】实施选定方案
```

### 3.5 同步更新机制

| 原则 | 说明 |
|------|------|
| **原子性** | 所有子模块同步更新，要么全成功要么全回滚 |
| **可追溯** | 每次更新记录版本、变更内容、变更原因 |
| **可回滚** | 每次更新前保存当前状态，支持一键回滚 |

### 3.6 Git提交规范

```bash
git commit -m "[EVOLUTION] 自主进化v2.0 - $(date +%Y-%m-%d)

## 变更摘要
- 调研模块: X个
- 优化方案: Y个
- 风险等级: Z"
```

## 5. 架构模式库（L3-R021）

### 5.1 80/20原则

**80%的使用场景应该保持单智能体。** 只有在明确需要时才添加多个智能体。

### 5.2 何时使用单智能体

- 问题范围狭窄且定义明确
- 偏好集中式推理
- 所有上下文适合单个上下文窗口

### 5.3 何时使用多智能体

- 任务需要专门的委派
- 单一视角不足
- 需要并行化或冗余
- 需要容错性

### 5.4 架构模式

| 模式 | 说明 |
|------|------|
| 层级模式 | 智能体按层级组织，高层监督低层 |
| 编排器-工作器 | 中央编排器分配任务给工作器 |
| 协作/群组聊天 | 多智能体参与共享对话线程 |
| 顺序/管道 | 智能体按管道组织，依次处理 |

## 6. 最佳实践库（L3-R022）

### 6.1 八大最佳实践

1. **分配明确角色** - 一个智能体专门负责研究，一个专门负责执行，一个专门负责验证

2. **保持内存本地** - 只保留最后几步的上下文，每个智能体只看到它需要的信息

3. **工具访问精细化** - 工具只开放给真正需要的智能体，保留调用日志

4. **模块化编排图** - 构建模块化和可扩展的编排图

5. **定期技能评估** - Curator系统持续评估技能性能

6. **智能保护机制** - 5级保护（none/low/medium/high/critical）

7. **故障恢复机制** - CircuitBreaker自动熔断和恢复

8. **自适应进化** - 基于成功率和历史调整蜂群规模
