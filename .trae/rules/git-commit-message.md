# Git提交信息规范

## 提交格式
```
<类型>(<范围>): <描述>

[可选的正文]

[可选的脚注]
```

## 提交类型
| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 错误修复 |
| docs | 文档变更 |
| style | 代码格式（不影响功能） |
| refactor | 重构（不是修复也不是新功能） |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具相关 |

## 规范要求
- 标题不超过50字符
- 使用中文描述
- 标题以动词开头
- 不使用句号结尾
- 正文解释what和why，不解释how

## 示例
```
feat(规则体系): 新增动态加载规则模块

- 支持根据上下文自动切换规则
- 减少不相关规则的干扰
- 提高规则复用率

Closes #123
```
