**版本**: v3.1 | **更新日期**: 2026-05-20 | **规则状态**: ✅ 生效中
**触发条件**: Git 操作或提交时自动加载
**L1支撑**: 交付标准
**L2支撑**: 质量规范

---

## 一、分支策略 [L3-3.1.1]

| 分支类型 | 命名规范 | 用途 | 保护 |
|----------|----------|------|------|
| main | main | 生产环境 | ✅ 保护 |
| 功能分支 | feat/xxx | 新功能开发 | - |
| 修复分支 | fix/xxx | Bug 修复 | - |
| 文档分支 | docs/xxx | 文档更新 | - |

---

## 二、提交信息规范

### 2.1 格式 [L3-3.2.1]
```
<type>(<scope>): <subject>

<body>  (可选)
<footer> (可选)
```

### 2.2 Type 类型 [L3-3.2.2]
| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档更新 |
| style | 代码格式调整 |
| refactor | 重构 |
| test | 测试相关 |
| chore | 构建/工具 |

### 2.3 示例 [L3-3.2.3]
```git
feat(auth): add JWT token validation

- add token verification middleware
- add unit tests for auth service
- update API documentation

Closes #123
```

## 三、提交前检查清单 [L3-3.3.1]

- [ ] 测试通过
- [ ] 静态检查通过 (lint)
- [ ] 代码格式化完成
- [ ] 无遗留调试代码
- [ ] 提交信息符合规范
