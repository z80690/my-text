# Git 工作流 (L3)

**触发条件**：当对话涉及 `git` 操作或提交时自动加载。

## 提交信息格式
```
<type>(<scope>): <subject>

[可选正文]

[可选脚注]
```

### 提交类型
| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 错误修复 |
| docs | 文档变更 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具相关 |

## 工作流程
- 提交前必须运行测试和 lint
- 分支策略：`main` 保护，功能开发在 `feat/` 分支

## 规范要求
- 标题不超过50字符
- 使用中文描述
- 标题以动词开头
- 不使用句号结尾
