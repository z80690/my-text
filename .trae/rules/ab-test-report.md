## 一、测试场景设计 [L3-10.1.1]

### 测试场景：开发一个用户认证模块

**任务描述**：
```
需求：开发一个用户认证模块，包含登录、注册、密码找回三个功能
要求：
- 用户可以注册账号（邮箱+密码）
- 用户可以登录系统
- 用户可以找回密码（发送重置邮件）
- 安全性要求：不能明文存储密码
- 需要单元测试
```

## 四、测试结论

### 4.1 能力提升总结 [L3-10.4.1]

| 提升类型 | 具体表现 |
|----------|----------|
| **新增能力** | 质量门禁、任务拆分规范、测试指南、规则冲突解决 |
| **显著提升** | 质量保障(+200%)、测试(+300%)、规则管理(+200%) |
| **体系完善** | 从碎片化规则到完整的三层架构 |

### 4.2 保持不变的能力 [L3-10.4.2]

| 能力项 | 说明 |
|--------|------|
| 根本身份 | 仍是全栈开发协作者 |
| 协作铁律 | 仍强调需求对齐、方案对比、环境感知 |
| 安全铁律 | 仍是最高优先级 |

### 4.3 潜在风险 [L3-10.4.3]

| 风险项 | 等级 | 缓解措施 |
|--------|------|----------|
| 规则复杂度增加 | 低 | 提供触发条件，只在需要时加载 |
| 执行成本增加 | 中 | 简化流程，聚焦核心检查项 |

### 4.4 最终结论 [L3-10.4.4]

✅ **现有规则体系(v3.x)在各项能力上均有提升或保持，无能力下降**。
