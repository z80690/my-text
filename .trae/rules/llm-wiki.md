## 一、Ingest操作执行细则

### 1.1 触发条件
- 新的用户输入包含重要知识
- 外部调研获取新信息
- 项目文件有重要变更
- 明确要求"编译到wiki"

### 1.2 执行步骤
```
1. 识别知识类型 → 确定页面类型
2. 提取核心信息 → 结构化整理
3. 生成wiki页面 → 符合页面规范
4. 建立双向链接 → 关联相关页面
5. 更新index.md → 添加到索引
```

### 1.3 页面类型判断

| 知识类型 | 对应页面 |
|---------|---------|
| 人物、组织、项目 | Entities |
| 技术、理论、概念 | Concepts |
| 文档、视频、资料摘要 | Summaries |
| 分析、对比、综合 | Synthesis |
| 常见问题、FAQ | Queries |

### 1.4 质量检查
- [ ] 页面结构完整
- [ ] 核心信息齐全
- [ ] 相关链接有效
- [ ] 标签分类正确

---

## 二、Query操作执行细则

### 2.1 查询优先级
1. **Wiki优先**：先检查wiki是否已有答案
2. **索引辅助**：通过index.md快速定位
3. **双向链接**：利用关联页面扩展答案
4. **按需编译**：wiki没有的内容才去原始资料

### 2.2 查询流程
```
用户查询 → 检查wiki → 有答案直接返回
              ↓ 没有
         去原始资料 → 编译成wiki → 返回答案
```

### 2.3 Wiki检查清单
- [ ] 检查index.md索引
- [ ] 检查相关概念页面
- [ ] 检查相关问答页面
- [ ] 检查综合分析页面

---

## 三、Lint操作执行细则

### 3.1 检查周期
- 每次新增/更新页面后立即检查
- 每周进行一次全面检查

### 3.2 检查项
- **页面完整性**：结构完整、信息齐全
- **链接有效性**：双向链接正确、无死链
- **内容一致性**：无矛盾、无重复
- **知识时效性**：过时内容标记、需要更新
- **标签规范性**：分类正确、标签统一

### 3.3 自动修复
- 简单格式问题自动修复
- 链接错误自动修正
- 重复内容自动合并

## 五、细粒度切块拆分

### 5.1 触发条件
- 文档长度超过1000字符
- 包含两个及以上二级标题
- 收到切块存储指令
- 系统自动判定长文档结构

### 5.2 核心切分原则
块大小控制在300-500字符，相邻内容保留10%-15%文字重叠规避语义断裂；优先依照标题层级切割，不在语句、表格、代码中间拆分内容。

### 5.3 分级拆分存储
一级父块存summaries目录，留存全文框架与核心结论，作为检索总入口；
二级章节块存concepts目录，单二级标题独立成文件，承载完整章节信息；
三级细粒度叶子块存concepts目录，按语义拆分最小独立内容单元，用于细节查询。

### 5.4 差异化切割标准
技术文档跟随标题层级拆分，图表公式完整保留；对话内容按问答轮次划分；表格整块存储不拆解；代码以函数代码块为最小切割单位。

### 5.5 索引链接硬性规范
父块末尾挂载全部子块跳转链接，子块头部标注所属上级文档、层级标签；全局总索引同步更新各级知识块入口，实现上下级互相关联跳转。

### 5.6 检索调取顺序
优先匹配细粒度叶子块查询细节，其次调取二级章节块查看段落内容，再读取一级摘要了解整体，最终溯源原始文档补充未收录信息。

### 5.7 切块验收标准
单块字符合规、内容语义连贯无割裂；特殊格式内容完好无损；上下级链接全部有效；全局索引同步更新完毕。
