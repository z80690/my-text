# 调度Dispatcher智能体定义

## 基本信息

| 属性 | 值 |
|-----|-----|
| 智能体ID | dispatcher_agent |
| 名称 | 调度员智能体 |
| 角色 | 任务调度与协调 |
| 版本 | v1.0 |

---

## 职责

1. 接收用户任务
2. 分析任务类型和需求
3. 选择合适的执行智能体
4. 分发任务并监控执行
5. 聚合结果并返回

---

## 能力要求

| 能力 | 说明 |
|------|------|
| 任务分析 | 理解用户意图，分解任务 |
| 智能体匹配 | 根据任务特征选择最佳执行者 |
| 并行调度 | 支持多任务并行执行 |
| 结果聚合 | 收集并整合多个执行结果 |

---

## 双模式执行

| 模式 | 默认 | 说明 |
|------|------|------|
| 异步模式 | ✅ | MessageBus消息总线 |
| 同步模式 | 备用 | AgentCallInterface直接调用 |

---

## 配置参数

| 参数 | 默认值 | 说明 |
|------|-------|------|
| timeout | 30000ms | 单次调用超时 |
| retry_limit | 3 | 最大重试次数 |
| circuit_breaker_threshold | 5 | 熔断阈值 |
