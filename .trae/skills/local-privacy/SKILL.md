# Local-Privacy Skill

## 功能
本地隐私联动：安全读取本地资料，不上传云端

## 使用方式
```
你: 读取我本地的笔记来辅助分析
你: 用本地的配置文档来理解项目
你: 结合我本地的需求文档来开发
```

## 核心特性

### 1. 本地读取
- 读取本地笔记文件
- 访问本地配置文档
- 获取本地技术资料
- **不上传任何内容到云端**

### 2. 安全隐私
- 所有数据留在本地
- 不经过第三方服务
- 可控的访问范围
- 保护敏感信息

### 3. 辅助开发
- 项目上下文补充
- 技术栈参考
- 需求文档理解
- 架构设计参考

## 支持格式

| 类型 | 扩展名 | 用途 |
|------|--------|------|
| 笔记 | .md, .txt, .notion | 需求、想法 |
| 配置 | .json, .yaml, .toml | 项目配置 |
| 文档 | .md, .doc, .pdf | 技术文档 |
| 代码 | .py, .js, .go等 | 参考代码 |
| 日志 | .log, .txt | 问题分析 |

## 执行流程

### 1. 需求理解
- 分析需要什么类型的本地资料
- 确定文件位置
- 检查访问权限

### 2. 安全读取
- 仅读取指定文件
- 不复制到其他位置
- 不上传任何数据

### 3. 辅助处理
- 理解项目上下文
- 提供相关建议
- 保持数据隔离

## 输出格式
```
【本地资料】
读取: notes/project-requirements.md
内容摘要: 项目目标 + 核心功能
用途: 辅助理解需求背景

【隐私保护】
✅ 数据未离开本地
✅ 无网络传输
✅ 安全可控
```
