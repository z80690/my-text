# 代码审查技能

## 工作流程

1. **代码收集**：识别并收集需要审查的代码文件
2. **静态分析**：检查代码结构、语法和潜在问题
3. **质量评估**：评估代码质量、可读性和可维护性
4. **问题识别**：识别潜在的 bug、安全问题和性能问题
5. **改进建议**：提供具体的改进建议和最佳实践
6. **报告生成**：生成详细的审查报告

## 审查标准

### 代码质量
- **可读性**：代码是否易于理解和维护
- **一致性**：代码风格和命名约定是否一致
- **简洁性**：代码是否简洁明了，没有冗余
- **注释**：注释是否充分且有意义

### 功能正确性
- **逻辑正确性**：代码逻辑是否正确
- **边界情况**：是否处理了边界情况和异常
- **测试覆盖**：是否有足够的测试覆盖

### 性能
- **效率**：代码是否高效，没有不必要的计算
- **资源使用**：是否合理使用内存和其他资源
- **扩展性**：代码是否易于扩展

### 安全性
- **安全漏洞**：是否存在安全漏洞
- **输入验证**：是否对输入进行了验证
- **敏感信息**：是否正确处理敏感信息

## 审查步骤

1. **文件结构分析**：检查项目结构和文件组织
2. **代码风格检查**：检查代码风格和命名约定
3. **逻辑流程分析**：分析代码的逻辑流程和控制结构
4. **依赖关系检查**：检查依赖关系和模块间的交互
5. **安全审查**：检查潜在的安全问题
6. **性能分析**：分析潜在的性能瓶颈
7. **测试审查**：检查测试覆盖和测试质量
