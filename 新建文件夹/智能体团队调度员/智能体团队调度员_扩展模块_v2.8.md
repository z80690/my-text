## 第三章 并行执行详解

### 3.1 概述

**100%并行调度**：所有任务默认使用并行执行模式，执行智能体完成任务，监控智能体实时监控。

### 3.2 工作流

```
任务输入→创建上下文→并行启动
              ┌──────┴──────┐
              ▼             ▼
        执行智能体      监控智能体
              │             │
              ├─步骤─┐      │
              ├─步骤─┼─→监控
              └─结果─┘      │
                      └→聚合→返回用户
```

### 3.3 协议

**启动指令**: `{"action":"parallel_start","task_id":"task_xxx","executor_agent":"code_executor_agent","monitor_agent":"monitor_agent","task":"...","context_id":"ctx_xxx"}`

**监控数据**: `{"monitor_id":"mon_xxx","task_id":"task_xxx","agent_id":"code_executor_agent","status":"running","start_time":"ISO8601","metrics":{"response_time_ms":1250,"progress":75,"errors":0,"logs":[...]}}`

## 第五章 路由策略详解

### 5.1 路由策略

| 策略 | 优先级 | 说明 |
|-----|-------|-----|
| 语义路由 | 最高 | 知识图谱意图分析 |
| 上下文路由 | 高 | 历史任务+用户偏好 |
| 负载路由 | 中 | 选择负载最低智能体 |
| 优先级路由 | 中 | 高优先级任务优先 |
| 并行路由 | **最高** | **所有任务自动配对监控智能体** |

### 5.2 路由算法

```python
def select_agent(task, context):
    if context.get("context_id"):
        ctx = get_context(context["context_id"])
        similar = ctx.query_similar_tasks(task)
        if similar and similar.agents_used:
            return similar.agents_used[0]
    if context.get("user_preference"):
        preferred = context["user_preference"].get("agent")
        if preferred and is_available(preferred):
            return preferred
    recommended = kg.recommend_agents(task)
    candidates = [a for a in agents if a.id in recommended]
    return min(candidates, key=lambda a: a.current_load)

def select_parallel_agents(task, context):
    return {"executor": select_agent(task, context), "monitor": "monitor_agent"}
```

### 5.3 复用策略

| 场景 | 策略 |
|-----|-----|
| 连续同类任务 | 复用智能体组合 |
| 用户偏好 | 优先使用偏好智能体 |
| 新任务 | 语义路由+记录历史 |
| 任务失败 | 切换备用智能体 |
| **所有任务** | **自动配对监控智能体** |

## 第六章 博弈调度详解

### 6.1 三种模式

| 模式 | 触发词 | 流程 |
|-----|-------|-----|
| 模式一：辩论 | 对比、权衡、优缺点 | 视角A→视角B→裁决 |
| 模式二：降维打击 | 优化、改进、重构 | 生成→挑剔→融合 |
| 模式三：深度设计 | 设计、架构、方案 | 蓝图→挑战→融合 |

### 6.2 触发流程

```
任务→分词→query_kg→概念聚合
    ├──含"辩论"→模式一
    ├──含"优化"→模式二
    ├──仅"设计"→模式三
    └──无概念→复杂度判断→模式三/常规路由
```

### 6.3 执行流程

**模式一**: 识别对立视角→视角A输出→视角B反对→综合裁决  
**模式二**: 创意生成→审查审阅→综合融合  
**模式三**: 架构蓝图→工程师评估→技术作家输出

### 6.4 优先级

- **博弈模块最高优先级**
- **所有任务默认并行执行（100%并行触发）**

```json
{"templates":{"parallel_execution":{"name":"并行执行","agents":["executor","monitor_agent"],"flow":[{"step":"start","agents":["executor","monitor"],"action":"init"},{"step":"execute","agent":"executor","action":"run"},{"step":"monitor","agent":"monitor","action":"track"},{"step":"complete","agent":"dispatcher","action":"aggregate"}]},"sequential_execution":{"name":"顺序执行","agents":["executor"],"flow":[{"step":"start","agent":"executor","action":"init"},{"step":"execute","agent":"executor","action":"run"},{"step":"complete","agent":"dispatcher","action":"finish"}]},"game_theory":{"name":"博弈模式","agents":["agent_a","agent_b","dispatcher"],"flow":[{"step":"perspective_a","agent":"agent_a","action":"analyze"},{"step":"perspective_b","agent":"agent_b","action":"oppose"},{"step":"arbitrate","agent":"dispatcher","action":"decide"}]}}}
```

### 7.2 工作流选择

| 任务类型 | 模板 |
|-----|-----|
| **所有任务** | **parallel_execution** |
| 博弈任务 | game_theory |
| 降级场景 | sequential_execution |

### 7.3 模板索引

| 模板ID | 模式类型 | 核心功能简述 |
|-------|---------|-------------|
| seq_flow | 顺序流 | 按顺序依次执行多个任务节点 |
| parallel_gateway | 并行网关 | 多个分支任务同时执行，等待全部完成后继续 |
| conditional_branch | 条件分支 | 根据条件表达式选择执行路径 |
| loop_flow | 循环 | 重复执行任务直到满足退出条件 |
| split_join | 分合流 | 将任务拆分为多个子任务并行执行，最后聚合结果 |

| game_theory | 博弈模式 | 多智能体辩论决策模式，支持三种博弈策略 |
| monitor_flow | 监控流 | 全程监控任务执行状态，实时收集性能指标 |

## 第八章 执行流程详解

### 8.1 执行步骤

```
用户输入→Step1:接收任务→Step2:图谱查询+上下文分析→Step3:工作流选择→Step4:并行调度→Step5:结果聚合→Step6:返回结果
```

**Step1**: 解析输入、关联/创建上下文、检查相似任务  
**Step2**: 分词查询、分析偏好、确定博弈模式  
**Step3**: 选择工作流模板（默认parallel_execution）  
**Step4**: 总线分发、启动监控、记录历史  
**Step5**: 收集输出、整合监控报告、更新记忆  
**Step6**: 输出结果、记录日志

### 8.2 错误处理

| 错误码 | 说明 | 处理 |
|-------|-----|-----|
| ERR_AGENT_UNAVAILABLE | 智能体不可用 | 切换备用 |
| ERR_CONTEXT_EXPIRED | 上下文过期 | 重新创建 |
| ERR_STATE_LOCKED | 状态锁定 | 等待/解锁 |
| ERR_ROUTE_FAILED | 路由失败 | 检查总线 |
| ERR_TIMEOUT | 执行超时 | 增加超时/终止 |
| ERR_MONITOR_FAILED | 监控失败 | 降级顺序执行 |

## 第九章 扩展能力

### 9.1 概述

系统支持通过外部引擎定义复杂工作流，涵盖顺序、分支、循环等控制模式，允许用户自定义任务编排逻辑，实现灵活的业务流程自动化。工作流引擎支持多种控制模式，包括顺序流、并行网关、条件分支、循环、分合流、补偿事务、博弈模式和监控流。

### 9.2 模板详情

| 模板ID | 模式类型 | 核心功能 |
|-------|---------|---------|
| seq_flow | 顺序流 | 按顺序执行任务 |
| parallel_gateway | 并行网关 | 多任务同时执行 |
| conditional_branch | 条件分支 | 根据条件选择路径 |
| loop_flow | 循环 | 重复执行直到满足条件 |
| split_join | 分合流 | 拆分执行后汇总 |
| compensation | 补偿事务 | 失败时回滚 |
| game_theory | 博弈模式 | 多智能体辩论决策 |
| monitor_flow | 监控流 | 全程监控执行状态 |
