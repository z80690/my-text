## 三、语义理解引擎

### 3.1 核心能力

向量化为文本生成向量；余弦相似度计算文本匹配度；意图识别确定任务类型；实体抽取提取关键信息。

### 3.2 向量相似度算法

```python
import math

class VectorMatcher:
    @staticmethod
    def cosine_similarity(v1: List[float], v2: List[float]) -> float:
        dot = sum(a * b for a, b in zip(v1, v2))
        n1, n2 = math.sqrt(sum(a*a for a in v1)), math.sqrt(sum(b*b for b in v2))
        return dot / (n1 * n2) if n1 and n2 else 0.0
```

### 3.3 意图识别

```python
class IntentRecognizer:
    INTENTS = {"code": ["写代码","debug","编程"], "design": ["设计","架构"], "review": ["审核","检查"], "research": ["调研","研究"]}

    def recognize(self, text: str) -> str:
        for intent, kws in self.INTENTS.items():
            if any(kw in text for kw in kws):
                return intent
        return "general"
```

## 四、知识图谱系统

### 4.1 核心结构

```python
class KnowledgeGraph:
    def __init__(self):
        self.nodes = {}
        self.edges = []

    def add_node(self, id: str, type: str, props: Dict):
        self.nodes[id] = {"type": type, "properties": props}

    def add_edge(self, src: str, tgt: str, rel: str):
        self.edges.append({"source": src, "target": tgt, "relation": rel})
```

### 4.2 节点与关系

| 关系类型 | 说明 |

### 4.3 图谱查询

def find_agents_by_skill(kg: KnowledgeGraph, skill: str) -> List[str]:
    return [nid for nid, n in kg.nodes.items() if n["type"] == "agent" and skill in n["properties"].get("skills", [])]

### 6.3 上下文保持规则

## 九、固定团队机制

3个预设团队，有明确分工和工作流程。

### 9.1 开发团队 (dev_team)

- **负责人**: dispatcher_agent
- **角色**: 项目经理、需求分析、架构设计、代码开发、测试验证、部署发布
- **流程**: 需求分析→架构设计→代码开发(可并行)→测试验证→部署发布

- **适用**: 研究、调研、分析

### 9.3 审核团队 (review_team)

- **负责人**: monitor_agent
- **角色**: 审核监督、内容审核、质量审核、最终审核
- **流程**: 初审→复审→终审→监督报告
