## 三、规则引擎统一体系（v3.6核心改进）

### 3.1 规则与智能体整合

**问题**: v3.5中规则引擎(rule_engine.py)与智能体系统(BaseAgent)是独立实现的两套体系

**解决方案**: 让规则引擎继承BaseAgent，形成统一继承体系

```python
# 整合后的规则基类
class BaseRule(BaseAgent):
    """规则基类 - 继承自BaseAgent，统一体系"""

    def __init__(self, rule_id: str, name: str, level: str, severity: str):
        config = AgentConfig(
            id=rule_id,
            name=name,
            type='rule',
            capabilities=['evaluate', 'execute']
        )
        super().__init__(config)
        self._level = level
        self._severity = severity

    def _default_execute(self, task: str, context: dict) -> dict:
        """规则执行逻辑"""
        rule_context = RuleContext(**context)
        if self.evaluate(rule_context):
            return self.execute(rule_context)
        return {'status': 'skipped'}

    @abstractmethod
    def evaluate(self, context: RuleContext) -> bool:
        """评估条件"""
        pass

    @abstractmethod
    def _do_execute(self, context: RuleContext) -> Any:
        """执行动作"""
        pass
```

│  - 继承: L1 + L2的所有约束                                  │

│  - 示例: R001工具优先规则、R002自动记忆规则                   │

class VectorMatcher:
    def cosine_similarity(v1, v2):
        dot = sum(a*b for a,b in zip(v1,v2))
        n1, n2 = math.sqrt(sum(a*a for a in v1)), math.sqrt(sum(b*b for b in v2))
        return dot / (n1*n2) if n1 and n2 else 0.0
