# 代码执行智能体增强规则 v2.1
*字数：5,600字*

## 一、新增核心能力
1. 短期上下文管理：自动关联对话消息
2. 长期知识记忆：三层记忆系统（工作/情景/语义）
3. 统一状态管理：state操作协议
4. 记忆增强路由：基于历史选择智能体

## 二、中央总线增强协议
**消息格式增强**：
来源 -> 目标 | 操作 | 消息ID | 上下文ID | 优先级 | {载荷} | {元数据}
操作类型：dispatch|query|result|error|state|memory
新增：state（状态操作）、memory（记忆操作）

**新增state操作**：
设置状态：agent -> bus | state | set | 路径 | 值
获取状态：agent -> bus | state | get | 路径
订阅状态：agent -> bus | state | subscribe | 路径
状态类型：运行时/配置/性能状态

**新增memory操作**：
保存工作记忆：agent -> bus | memory | save_working | 上下文ID | {数据}
读取情景记忆：agent -> bus | memory | load_episodic | 条件
查询语义记忆：agent -> bus | memory | query_semantic | 概念
清除记忆：agent -> bus | memory | clear | 类型

**记忆数据结构**：
{
  "id": "mem_001",
  "entity": "实体标识",
  "type": "preference|fact|procedure|outcome",
  "content": {},
  "tags": [],
  "access_count": 数字,
  "created_at": "时间戳"
}

## 三、三层记忆系统
**工作记忆**：当前对话上下文，自动清理（24小时）
**情景记忆**：任务执行历史，可查询（保存30天）
**语义记忆**：长期知识库，持久化存储

## 四、上下文管理
**上下文ID格式**：ctx_时间戳_随机数
**生成规则**：新任务自动生成
**传播机制**：同一上下文ID的消息自动关联
**生命周期**：创建→使用→保存/清理

## 五、智能体协同增强
**记忆增强的调度流程**：
1. 接收任务，分析意图
2. 查询记忆：类似任务历史、用户偏好
3. 基于记忆选择最优智能体
4. 执行任务，保存过程
5. 任务完成，提取知识
6. 更新记忆，优化权重

**20个智能体的记忆能力**：
- code_review_agent: 学习代码规范
- testing_agent: 记忆测试模式
- docs_agent: 保存文档模板
- security_agent: 积累漏洞模式
- architecture_agent: 存储架构决策
- devops_agent: 记录部署历史
- 其他智能体：根据领域积累专业知识

## 六、技能库记忆支持
**12个核心技能的记忆关联**：
- caveman: 记忆用户沟通偏好
- design-an-interface: 记录设计决策
- domain-model: 保存模型版本
- git-guardrails: 学习团队规范
- github-triage: 积累分类规则
- improve-architecture: 存储架构决策
- migrate: 记录迁移历史
- refactor-plan: 保存重构模式
- scaffold: 记忆模板偏好
- pre-commit: 存储配置模板
- tdd: 记录测试模式
- zoom-out: 保存分析框架

**技能执行流程增强**：
技能触发 → 检查用户偏好 → 调整参数 → 查询历史 → 优化策略 → 保存结果 → 更新记忆

## 七、执行流程
**标准流程**：
1. 接收任务，生成上下文ID
2. 查询知识图谱和记忆
3. 选择调度模式（辩论/降维/深度）
4. 调度智能体执行
5. 聚合结果，保存工作记忆
6. 提取知识到语义记忆
7. 返回结果，清理工作记忆

**错误处理**：
- 智能体超时：重试，记录失败，降低权重
- 结果冲突：仲裁，保存冲突案例
- 参数错误：详细报错，记录错误模式
- 记忆满：清理旧记忆，提示用户

## 八、配置部署
**最小配置**：

**启动顺序**：
1. 启动中央总线
2. 连接知识图谱
3. 初始化记忆存储
4. 注册智能体
5. 启动健康检查
6. 接收任务

## 九、最佳实践
**上下文使用**：
1. 每个任务用新上下文ID
2. 单个上下文不超过50条消息
3. 任务完成及时清理工作记忆
4. 重要任务手动保存为情景记忆

**记忆管理**：
1. 重要知识存语义记忆，临时信息存工作记忆
2. 每月审查低价值记忆
3. 语义记忆每日备份
4. 高频记忆加缓存

**性能优化**：
1. 记忆查询异步执行
2. 常用字段加索引
3. 定期清理过期记忆
4. 监控命中率，调整缓存策略

**故障排查**：
- 记忆丢失：检查存储连接，恢复备份
- 响应慢：优化查询，添加索引
- 上下文混乱：确保上下文ID唯一
- 路由错误：清理污染记忆

## 十、使用示例
**示例1：多轮代码审查**
用户: "审查登录功能" → 生成ctx_001 → 查询记忆:用户偏好详细审查 → 选择详细模式 → 保存过程
用户: "再审查注册功能" → 使用ctx_001 → 记忆:用户偏好详细 → 继续详细审查 → 任务完成，提取偏好

**示例2：架构优化**
任务: "优化微服务架构" → 查询记忆:类似历史 → 发现"服务网格"成功 → 推荐类似方案 → 保存决策 → 如成功，提取为优化模式

**示例3：错误恢复**
智能体A超时 → 记录失败，降低A权重 → 选择B重试 → B成功 → 提高B权重 → 保存"任务适合B"

## 十一、快速参考
**核心命令**：
状态管理: agent -> bus | state | set/get/subscribe | 路径
记忆管理: agent -> bus | memory | save_working/load_episodic/query_semantic | 参数
上下文: metadata.context_id = "ctx_时间戳_随机数"

**错误代码**：
ERR_MEMORY_FULL: 记忆存储已满
ERR_CONTEXT_EXPIRED: 上下文过期
ERR_AGENT_UNAVAILABLE: 智能体不可用
ERR_ROUTE_FAILED: 路由失败
ERR_STATE_LOCKED: 状态被锁定

**监控指标**：
任务成功率(>95%)，平均响应时间(<3s)，记忆命中率(>70%)，上下文切换开销(<100ms)
