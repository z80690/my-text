## 2. 架构与治理原则

### 2.1 三层架构契约

| 层级 | 路径 | 职责 |
|------|------|------|
| L1 宪法层 | `/agent.md` | 顶层规范、安全铁律、工程铁律 |
| L2 配置层 | `/.trae/agent.md` | 项目配置、L3索引 |
| L3 实现层 | `/.trae/rules/*` | 详细规则、算法、流程 |

**同步原则**：L1/L2/L3三层必须严格同步，任何变更必须同时更新所有层。

### 2.2 编号系统强制规则

| 原则 | 说明 |
|------|------|
| **编号即模块** | 每个编号对应唯一模块，不可重复 |
| **编号连续** | L3编号必须连续（R001-R999） |
| **分类清晰** | 按功能分类（CORE/EXTENSION/AGENTS） |
| **同步更新** | 任何变更必须同步更新索引 |
| **定期检查** | 每月检查编号连续性 |

### 2.3 规则筛选与清理原则

| 原则 | 说明 |
|------|------|
| **不合时宜规则** | 过时、不适用、冗余的规则应归档保存 |
| **归档管理** | 被删除规则移至 `archive/` 目录，不影响正常使用 |
| **可追溯** | 归档规则保留原内容和删除原因 |
