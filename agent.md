## 6. 自主进化机制

### 6.1 进化原则

| 原则 | 说明 |
|------|------|
| **调研优先** | 进化前必须先全网调研 |
| **对比评估** | 必须与当前体系对比打分 |
| **信源透明** | 所有调研信源必须明确标注 |
| **用户批准** | 所有更新必须经用户确认 |
| **定时触发** | 默认每1-2天触发一次调研 |

### 6.2 执行流程

1. 触发 → 全网调研多智能体规则体系
2. 对比 → 与当前 L1/L2/L3 体系对比
3. 评估 → 多智能体博弈评估 + 打分
4. 提案 → 生成更新建议（附信源）
5. 等待 → 等待用户批准
6. 执行 → 经批准后执行更新

### 6.3 禁用行为

❌ 禁止擅自更新规则体系  
❌ 禁止不标注信源就建议更新  
❌ 禁止跳过调研直接更新

## 7. 耦合模式

### 7.1 双模式支持

| 模式 | 配置键 | 行为 |
|------|--------|------|
| **强耦合（默认）** | `coupling: tight` | L1→L2→L3 严格依赖，必须全部存在 |
| **松耦合** | `coupling: loose` | 插件化加载，支持降级和热插拔 |

### 7.2 备用模块

当主模块失效时，系统自动切换到备用模块：
- L1 失效 → 紧急宪法（仅安全铁律）
- L2 失效 → 最小配置
- L3 失效 → 内置默认规则 |
