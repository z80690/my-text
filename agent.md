## 2. 三层双向同步规则

### 2.1 三层架构

| 层级 | 路径 | 角色 | 职责 |
|------|------|------|------|
| L1 | `/agent.md` | 顶层规范/契约 | 定义核心原则、规则框架 |
| L2 | `/.trae/agent.md` | 项目配置/约束 | 项目级配置、融合策略 |
| L3 | `/.trae/rules/*` | 具体实现/规则 | 详细规则、领域特定规范 |

### 2.2 同步原则

| 原则 | 说明 | 执行策略 |
|------|------|---------|
| L1 有 → L2必须有 → L3必须有 | 向下实现 | 自动同步 |
| L1 没有 → L2不能有 → L3不能有 | 禁止凭空创建 | 校验拦截 |
| 冲突时 → L1为准 | 顶层优先 | 自动修正 |

## 3. 新老规则体系融合方法

### 3.1 融合策略矩阵

| 融合场景 | 老规则体系 | 新规则体系 | 融合策略 | 优先级 |
|---------|-----------|-----------|---------|--------|
| **核心原则一致** | 有 | 有 | 保留并整合 | P0 |
| **老规则更详细** | 详细 | 简略 | 迁移细节到L3 | P1 |
| **新规则更完善** | 简略 | 详细 | 采用新规则 | P1 |
| **规则冲突** | 存在 | 存在 | 以新规则为准 | P0 |
| **老规则独有** | 有 | 无 | 评估后决定保留或淘汰 | P2 |
| **新规则独有** | 无 | 有 | 直接采用 | P1 |

### 3.2 融合流程

| 步骤 | 动作 | 说明 | 负责层级 |
|------|------|------|---------|
| 1 | 规则识别 | 识别新老规则体系中的所有规则 | L1 |
| 2 | 规则比对 | 逐规则对比，标记差异和冲突 | L1 |
| 3 | 冲突解决 | 根据融合策略解决冲突 | L1 |
| 4 | 规则迁移 | 将详细规则迁移到L3 | L3 |
| 5 | 配置更新 | 更新L2配置文件 | L2 |
| 6 | 验证同步 | 验证三层规则一致性 | 全层级 |

### 3.3 规则分类与处理

| 规则类别 | 处理方式 | 目标位置 | 说明 |
|---------|---------|---------|------|
| 核心原则 | 保留整合 | L1 | 只保留核心，去重合并 |
| 详细规范 | 迁移细化 | L3 | 详细实现放到L3 |
| 项目配置 | 更新同步 | L2 | 项目特有配置 |
| 废弃规则 | 标记淘汰 | - | 记录原因后移除 |
| 冲突规则 | 以新为准 | L1/L2/L3 | 新规则覆盖老规则 |

### 3.4 融合决策树

```
规则存在于老体系？
    ├─ 否 → 新规则独有 → 直接采用 → 更新L1/L2/L3
    └─ 是 → 规则存在于新体系？
        ├─ 否 → 老规则独有 → 评估价值 → 保留或淘汰
        └─ 是 → 规则内容一致？
            ├─ 是 → 保留整合 → 去重合并
            └─ 否 → 规则冲突 → 以新规则为准
```

## 4. 工具优先原则

智能体应优先使用工具完成任务，而非依赖内部知识。

| 原则 | 说明 |
|------|------|
| 信息获取 | 能用工具查，不用内部知识 |
| 操作执行 | 能用工具做，不手动编写 |
| 工具是唯一合法途径 | 智能体与外部世界交互的标准方式 |

### 9.2 分诊流程

| 步骤 | 名称 | 动作 |

## 14. 分层编号体系

### 14.1 编号格式

| 组成部分 | 说明 | 示例 |
|---------|------|------|
| L{层级} | 层级标识 | L1、L2、L3 |
| -{类型} | 类型编码 | A、B、C、R、S |
| {序号} | 顺序编号 | 001、002、003 |
| [-{内部编号}] | 可选内部编号 | -01、-02 |

### 14.2 类型编码

| 编码 | 类型 | 说明 |
|------|------|------|
| A | Abstract | 抽象定义、顶层契约 |
| B | Base | 基础实现、配置层 |
| C | Concrete | 具体对象、智能体 |
| R | Rule | 规则文件 |
| S | Skill | 技能模块 |

### 14.3 示例

| 编号 | 说明 |
|------|------|
| L1-A001 | 第一层，抽象类，第1个 |
| L2-B001 | 第二层，接口实现，第1个 |
| L3-C001-01 | 第三层，具体对象，第1个，内部第1节 |
