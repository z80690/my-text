## 2. 架构与治理原则

### 2.1 三层架构契约

| 层级 | 路径 | 职责 | 文件数量 |
|------|------|------|----------|
| L1 宪法层 | `/agent.md` | 顶层规范、安全铁律、工程铁律 | 1个 |
| L2 配置层 | `/.trae/agent.md` | 项目配置、Skills索引、Agents定义 | 1个 |
| L3 实现层 | `/.trae/rules/*.md` | 核心实现（聚合为2-3个文件） | 2-3个 |

**同步原则**：L1/L2/L3三层必须严格同步，任何变更必须同时更新所有层。

### 2.2 智能体核心定义

| 组件 | 说明 |
|------|------|
| **Dispatcher** | 中央调度智能体，负责任务分发和协调 |
| **Router** | 智能路由引擎，负责意图识别和路由 |
| **Swarm** | 蜂群协作引擎，负责多智能体协作 |
| **Memory** | 记忆系统，负责上下文管理 |

### 2.3 核心调度机制

| 模式 | 触发条件 | 执行方式 |
|------|----------|----------|
| **同步模式** | 默认 | AgentCallInterface直接调用 |
| **异步模式** | 消息总线 | MessageBus异步消息传递 |
| **蜂群模式** | 复杂任务 | 去中心化协作 |

## 3. 安全铁律（绝对底线）

| 序号 | 铁律 | 说明 |
|------|------|------|
| S1 | 不泄露凭据 | 任何情况下不泄露API Key、Token等 |
| S2 | 不绕过验证 | 不绕过安全验证和授权机制 |
| S3 | 不执行未授权删除 | 不删除未备份的重要文件 |
| S4 | 不暴露隐私 | 不记录或暴露用户敏感信息 |

## 4. 工程铁律

| 序号 | 铁律 | 说明 |
|------|------|------|
| E1 | 工具优先 | 优先使用工具而非纯文本输出 |
| E2 | 原子性变更 | 多个相关变更必须同时成功或回滚 |

## 5. 编号系统强制规则

| 原则 | 说明 |
|------|------|
| **编号即模块** | 每个编号对应唯一模块，不可重复 |
| **编号连续** | L3编号必须连续（R001-R999） |
| **分类清晰** | 按功能分类（core/memory/skills） |
| **同步更新** | 任何变更必须同步更新索引 |
| **定期检查** | 每月检查编号连续性 |

## 6. 规则筛选与清理原则

| 原则 | 说明 |
|------|------|
| **不合时宜规则** | 过时、不适用、冗余的规则应归档保存 |
| **归档管理** | 被删除规则移至 `archive/` 目录，不影响正常使用 |
| **可追溯** | 归档规则保留原内容和删除原因 |
