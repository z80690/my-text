
### 3.2 核心技术原理

#### 3.2.1 通讯协议模拟

检测仪通过**模拟内外机通讯协议**实现故障诊断：

| 协议类型 | 说明 |
|----------|------|
| **半双工串行通讯** | 最常用，帧格式为NRZ（不归零） |
| **差分信号传输** | 抗干扰能力强，适合长距离 |
| **RS485/CAN总线** | 部分高端机型支持 |

#### 3.2.2 信号采集原理

```
空调主板 → 通讯线(S) → 光耦隔离 → MCU解析 → 故障代码/运行参数
    ↓
  L/N电源线 → 电压/电流检测 → ADC转换 → 数据处理
```

#### 3.2.3 核心检测参数

| 参数类型 | 检测内容 | 正常范围 |
|----------|----------|----------|
| **电压** | 直流母线电压 | 310V±20V |
| **电流** | 压缩机运行电流 | 额定电流1.5-2倍（启动） |
| **频率** | 压缩机运行频率 | 30-120Hz |
| **温度** | 排气/回气/盘管温度 | 排气70-90℃ |
| **压力** | 高压/低压压力 | 高压1.5-2.0MPa |

### 3.3 软件实现架构

#### 3.3.1 MCU程序结构

```c
// 主程序框架（简化版）
void main() {
    system_init();         // 系统初始化
    protocol_select();     // 选择通讯协议
    
    while(1) {
        if (comm_ready) {
            send_query();      // 发送查询命令
            receive_data();    // 接收主板数据
            parse_params();    // 解析运行参数
            check_fault();     // 故障诊断
            display_result();  // 显示结果
        }
    }
}
```

#### 3.3.2 通讯帧结构（典型9600bps）

| 字节 | 内容 | 说明 |
|------|------|------|
| 1 | 0x7E | 帧头 |
| 2 | 地址 | 内机/外机地址 |
| 3 | 命令码 | 查询/控制命令 |
| 4-N | 数据 | 参数数据 |
| N+1 | 校验和 | CRC/XOR校验 |
| N+2 | 0x7F | 帧尾 |

#### 3.3.3 故障诊断算法

```python
def diagnose_fault(params):
    faults = []
    
    # 电压检测
    if params['dc_bus_voltage'] < 260:
        faults.append('E1: 欠压保护')
    elif params['dc_bus_voltage'] > 380:
        faults.append('E2: 过压保护')
    
    # 温度检测
    if params['exhaust_temp'] > 100:
        faults.append('P4: 排气温度过高')
    
    # 通讯检测
    if params['comm_status'] == 'error':
        faults.append('E6: 通讯故障')
    
    # 压缩机检测
    if params['compressor_freq'] < 30:
        faults.append('不升频故障')
    
    return faults
```

## 六、使用方法与操作流程

### 6.1 接线方法

#### 6.1.1 外机检测接线

```
检测仪端子        空调外机端子
    L ────────────── L（火线）
    N ────────────── N（零线）
    S ────────────── S（通讯线）
```

**注意**：检测外机时需断开内机通讯线！

#### 6.1.2 内机检测接线

```
检测仪端子        空调内机端子
    L ────────────── L（火线）
    N ────────────── N（零线）
    S ────────────── S（通讯线）
```

**注意**：检测内机时需断开外机通讯线！

### 6.2 操作流程

```
Step 1: 断电 → 按接线图连接检测仪
Step 2: 上电 → 等待检测仪红灯亮起（10-20秒）
Step 3: 扫码 → 手机扫描二维码进入小程序
Step 4: 选模式 → 选择检测模式（整机/内机/外机）
Step 5: 选品牌 → 选择对应空调品牌
Step 6: 读数据 → 查看运行参数和故障代码
Step 7: 断电 → 断电重启切换协议（如通讯失败）
```

### 6.3 故障诊断示例

| 故障代码 | 含义 | 可能原因 | 解决方案 |
|----------|------|----------|----------|
| E1 | 室温传感器故障 | 传感器开路/短路 | 更换传感器 |
| E6 | 通讯故障 | 通讯线断/主板损坏 | 检查线路/更换主板 |
| P4 | 排气温度保护 | 缺氟/散热不良 | 加氟/清洗外机 |
| H5 | 模块保护 | 压缩机过载 | 检查压缩机 |

#### 7.2.2 故障诊断算法

```c
// 故障诊断函数
void diagnose_fault(ac_params_t *params, fault_list_t *faults) {
    // 电压检测
    if (params->dc_voltage < 260) {
        add_fault(faults, FAULT_UNDERVOLTAGE);
    }
    
    // 温度检测
    if (params->exhaust_temp > 100) {
        add_fault(faults, FAULT_HIGH_TEMP);
    }
    
    // 通讯检测
    if (params->comm_error_count > 3) {
        add_fault(faults, FAULT_COMM);
    }
    
    // 频率检测
    if (params->target_freq > params->actual_freq + 10) {
        add_fault(faults, FAULT_FREQ_MISMATCH);
    }
}
```
