# Cloud Code 记忆系统 - AI行为规则 v1.0

## 核心定义

**自动** = AI收到用户消息后，**无需任何触发**，自动完成所有记忆处理

---

## AI标准行为流程（每次回复必须执行）

```
用户消息 → 【自动记忆处理】（静默） → 回复用户
```

### 【自动记忆处理】步骤

当AI收到用户消息时，**必须自动**执行：

#### 第一步：暗知识识别（自动）

分析用户消息，判断是否为暗知识：

| 暗知识特征 | 示例 | 处理方式 |
|-----------|------|---------|
| 个人偏好 | "我习惯..."、"我喜欢..." | 记忆 |
| 团队规则 | "我们团队禁止..." | 记忆 |
| 未来指令 | "以后都这样..." | 记忆 |
| 设计原因 | "因为要兼容..." | 记忆 |

#### 第二步：自动分类

```
包含"好"、"认可" → feedback/
包含文档路径 → reference/
包含项目、设计 → project/
其他个人偏好 → user/
```

#### 第三步：自动写入

**立即创建记忆文件**

#### 第四步：互斥检测（自动）【RULE-001 强制执行】

**⚠️ 两套规则体系绝对不能混用！**

在执行任何任务前，**必须检测**：

```
检测任务中的路径引用：
  ├── 仅包含 .opencode/  → ✅ 允许，使用新规则体系
  ├── 仅包含 .trae/      → ❌ 拒绝，这是旧体系
  └── 同时包含两者      → 🚫 严重违规！立即终止任务

违规处理：
  1. 立即输出警告："🚫 检测到规则体系混用！已阻止任务执行"
  2. 记录违规到记忆文件
  3. 不执行任何任务
  4. 要求用户明确指定使用哪套体系
```

**路径白名单**：
- `.opencode/` → 新规则体系（L1/L2/L3）
- `.trae/` → 旧规则体系（保持原样）

**任何混用都是严重违规！**
