# Cloud Code 记忆系统自动执行规则 v1.0

## 核心机制

**触发条件**: AI收到用户消息后，**自动**执行记忆处理，无需用户手动调用

---

## 第一章：自动记忆触发规则

### 1.1 触发时机

每次用户消息后，AI**必须自动**执行以下流程：

```
用户消息 → 自动识别暗知识 → 自动分类 → 自动写入记忆 → 必要时触发Dream整理
```

### 1.2 自动处理流程

#### 步骤1：暗知识识别
收到用户消息后，**立即**判断是否为暗知识：

```
IF 用户说了以下内容：
  - 个人偏好（我习惯、我喜欢、我不喜欢）
  - 团队规则（我们团队、禁止、必须）
  - 未来指令（以后、以后都这样、以后不要）
  - 设计原因（因为要、为了兼容、历史遗留）
  - 项目背景（这是X系统、我们项目）
  - 外部引用（Jira票号、文档位置）

THEN:
  标记为"暗知识"
  自动写入对应记忆目录
```

#### 步骤2：自动分类
```
IF 包含"好"、"认可"、"以后都这样" → feedback/
IF 包含文档路径、工具链接 → reference/
IF 包含项目、设计、原因 → project/
ELSE → user/
```

#### 步骤3：自动写入
```
自动创建文件：.trae/memories/{type}/{简化标题}.md
格式：
---
type: {类型}
created: {日期}
---

# {标题}

{原文内容}

**Why:** 自动识别的理由
**How to apply:** 根据类型触发
```

#### 步骤4：Dream自动整理
```
IF 距上次整理 ≥ 24小时 AND 会话数 ≥ 5:
  自动执行：扫描 → 整合 → 剪枝 → 生成报告
```

## 第二章：记忆自动调用API

### 2.1 核心函数（由AI自动调用）

```python
class AutoMemory:
    def process(self, user_message: str) -> dict:
        """
        AI收到消息后自动调用此方法
        无需用户触发，完全自动
        """
        # 1. 识别暗知识
        is_dark, why = self.is_dark_knowledge(user_message)

        if is_dark:
            # 2. 自动分类
            mem_type = self.classify(user_message)

            # 3. 自动写入
            result = self.write(user_message, mem_type)

            # 4. 检查Dream
            self.check_dream()

            return {"auto_saved": True, "type": mem_type, "path": result}

        return {"auto_saved": False}
```

### 2.2 触发关键词

| 关键词类型 | 示例 | 记忆类型 |
|-----------|------|---------|
| 个人偏好 | "我习惯4空格" | user |
| 团队规则 | "禁止用for循环" | project |
| 认可反馈 | "很好，以后都这样" | feedback |
| 设计原因 | "为了兼容旧客户端" | project |
| 项目背景 | "这是电商系统" | project |
| 文档位置 | "API文档在docs/" | reference |
| Jira引用 | "需求在ABC-123" | reference |

## 第三章：自动执行要求

### 3.1 AI必须自动执行
- **禁止**等待用户说"帮我记住"
- **禁止**等待用户手动触发
- **必须**在收到消息后立即处理

### 3.2 自动化程度
- 100%自动识别
- 100%自动分类
- 100%自动写入
- 100%自动Dream整理

### 3.3 响应格式
```
AI收到消息后：
1. 正常回复用户
2. 同时自动完成记忆处理
3. 不需要告诉用户"已保存到记忆"

（静默模式，记忆处理对用户透明）
```
