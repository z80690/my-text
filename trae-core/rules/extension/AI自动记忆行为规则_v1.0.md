# Cloud Code 记忆系统 - AI行为规则 v1.0

## 核心定义

**自动** = AI收到用户消息后，**无需任何触发**，自动完成所有记忆处理

---

## AI标准行为流程（每次回复必须执行）

```
用户消息 → 【自动记忆处理】（静默） → 回复用户
```

### 【自动记忆处理】步骤

当AI收到用户消息时，**必须自动**执行：

#### 第一步：暗知识识别（自动）

分析用户消息，判断是否为暗知识：

| 暗知识特征 | 示例 | 处理方式 |
|-----------|------|---------|
| 个人偏好 | "我习惯..."、"我喜欢..." | 记忆 |
| 团队规则 | "我们团队禁止..." | 记忆 |
| 未来指令 | "以后都这样..."、"以后不要..." | 记忆 |
| 设计原因 | "因为要兼容..."、"历史遗留..." | 记忆 |
| 项目背景 | "这是...系统"、"我们项目..." | 记忆 |
| 外部引用 | "Jira票号..."、"文档在..." | 记忆 |

| 明知识特征 | 示例 | 处理方式 |
|-----------|------|---------|
| 版本号 | "用的是React 18.2.0" | **不记忆** |
| 文件位置 | "在utils.js里..." | **不记忆** |
| 函数名 | "有个formatDate函数" | **不记忆** |

#### 第二步：自动分类

```
包含"好"、"认可"、"以后都这样" → feedback/
包含文档路径、工具链接 → reference/
包含项目、设计、原因、背景 → project/
其他个人偏好 → user/
```

#### 第三步：自动写入

**立即创建记忆文件**，文件路径格式：
```
.trae/memories/{type}/{标题}.md
```

文件格式：
```markdown
---
type: {user|feedback|project|reference}
created: {日期时间}
---

# {标题}

{用户原话}

**Why:** {为什么这是暗知识}
**How to apply:** {何时使用这条记忆}
```

#### 第四步：互斥检测（自动）【RULE-001 强制执行】

**⚠️ 两套规则体系绝对不能混用！**

在执行任何任务前，**必须检测**：

```
检测任务中的路径引用：
  ├── 仅包含 .trae/      → ✅ 允许，使用旧规则体系
  ├── 仅包含 .opencode/  → ❌ 拒绝，这是新体系
  └── 同时包含两者      → 🚫 严重违规！立即终止任务

违规处理：
  1. 立即输出警告："🚫 检测到规则体系混用！已阻止任务执行"
  2. 记录违规到记忆文件
  3. 不执行任何任务
  4. 要求用户明确指定使用哪套体系
```

**路径白名单**：
- `.trae/` → 旧规则体系（保持原样）
- `.opencode/` → 新规则体系（L1/L2/L3）

**任何混用都是严重违规！**

#### 第五步：静默执行

- **不告诉用户**"已保存到记忆"
- **不显示**任何记忆处理过程
- **完全透明**，用户无感知

---

## AI必须遵守的行为约束

### 禁止事项
- ❌ 禁止说"我会记住..."
- ❌ 禁止说"让我保存到记忆..."
- ❌ 禁止询问"要不要记住..."
- ❌ 禁止等待用户确认

### 必须事项
- ✅ 必须分析每条用户消息
- ✅ 必须自动识别暗知识
- ✅ 必须自动分类
- ✅ 必须自动写入记忆文件
- ✅ 必须静默执行

---

## 触发条件

**无条件触发**：每条用户消息都必须执行记忆分析

---

## 记忆类型说明

| 类型 | 目录 | 内容 |
|------|------|------|
| user | user/ | 用户个人偏好、习惯 |
| feedback | feedback/ | 用户确认、认可、反馈 |
| project | project/ | 项目背景、团队规则、设计决策 |
| reference | reference/ | 外部引用、文档位置、Jira票号 |
