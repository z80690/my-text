### 1.4 v2.9新增：状态机定义

```python
from typing import TypedDict, Optional, Annotated, List
from enum import Enum

class AgentStatus(Enum):
    IDLE = "idle"
    RUNNING = "running"
    COMPLETED = "completed"
    ERROR = "error"
    WAITING = "waiting"

class AgentState(TypedDict):
    messages: List[dict]              # 消息列表 [{role, content}]
    context: dict                     # 上下文数据
    status: str                       # 状态字符串
    result: Optional[dict]            # 执行结果
    execution_history: List[dict]     # 执行历史记录
    session_id: str                   # 会话ID
    created_at: str                   # 创建时间
    updated_at: str                   # 更新时间

def add_messages(left: list, right: list) -> list:
    """状态聚合器：合并消息列表"""
    return left + right
```

## 第三章 并行执行详解

### 3.1 概述

**100%并行调度**：所有任务默认使用并行执行模式。

### 3.2 工作流

```
任务输入→创建上下文→并行启动
              ┌──────┴──────┐
              ▼             ▼
        执行智能体      监控智能体
              │             │
              ├─步骤─┐      │
              ├─步骤─┼─→监控
              └─结果─┘      │
                      └→聚合→返回用户
```

### 3.3 协议

**启动指令**: 
```json
{"action":"parallel_start","task_id":"task_xxx","executor_agent":"code_executor_agent","monitor_agent":"monitor_agent","task":"...","context_id":"ctx_xxx"}
```

**监控数据**: 
```json
{"monitor_id":"mon_xxx","task_id":"task_xxx","agent_id":"code_executor_agent","status":"running","start_time":"ISO8601","metrics":{"response_time_ms":1250,"progress":75,"errors":0,"logs":[...]}}
```

## 第五章 路由策略详解

### 5.1 路由策略

| 策略 | 优先级 | 说明 |
|-----|-------|-----|
| 语义路由 | 最高 | 知识图谱意图分析 |
| 上下文路由 | 高 | 历史任务+用户偏好 |
| 负载路由 | 中 | 选择负载最低智能体 |
| 优先级路由 | 中 | 高优先级任务优先 |
| 并行路由 | 最高 | 所有任务自动配对监控智能体 |

def select_agent(task, context):
    if context.get("context_id"):
        ctx = get_context(context["context_id"])
        similar = ctx.query_similar_tasks(task)
        if similar and similar.agents_used:

## 第六章 博弈调度详解（v2.9优化）

### 6.1 两种模式（简化）

| 模式 | 触发词 | 流程 |
|-----|-------|-----|
| **模式一：优化改进** | 优化、改进、重构、审查 | 生成→挑剔→融合 |
| **模式二：深度设计** | 设计、架构、方案、对比 | 蓝图→挑战→融合 |

**v2.9优化说明**：
- 原"模式一：辩论"合并到"模式二：深度设计"
- 对比分析属于设计任务的子集
- 简化后更易于实现和维护

### 6.2 触发流程

```
任务→分词→query_kg→概念聚合
    ├──含"优化/改进/重构"→模式一
    ├──含"设计/架构/对比"→模式二
    └──无概念→复杂度判断→模式二/常规路由
```

### 6.3 执行流程

**模式一**: 创意生成→审查审阅→综合融合  
**模式二**: 架构蓝图→工程师评估→技术作家输出

## 第八章 执行流程详解

### 8.1 执行步骤

```
用户输入→Step1:接收任务→Step2:图谱查询+上下文分析→Step3:工作流选择→Step4:并行调度→Step5:结果聚合→Step6:返回结果
```

### 8.2 错误处理

| 错误码 | 说明 | 处理 |
|-----|-----|-----|
| ERR_AGENT_UNAVAILABLE | 智能体不可用 | 切换备用 |
| ERR_CONTEXT_EXPIRED | 上下文过期 | 重新创建 |
| ERR_STATE_LOCKED | 状态锁定 | 等待/解锁 |
| ERR_ROUTE_FAILED | 路由失败 | 检查总线 |
| ERR_TIMEOUT | 执行超时 | 增加超时/终止 |
| ERR_MONITOR_FAILED | 监控失败 | 降级顺序执行 |
