# 高级工作流模板索引 v3.0

## 概述

本文档为智能体团队调度员 v3.0 的可扩展工作流引擎提供完整的模板索引，包含所有支持的工作流模式定义。继承 v2.7 所有模板并新增企业级特性。

---

## 模板索引

| 模板ID | 模式类型 | 核心功能简述 | QoS级别 | 分布式支持 |
|-------|---------|-------------|--------|-----------|
| seq_flow | 顺序流 | 按顺序依次执行多个任务节点 | Normal | ✅ |
| parallel_gateway | 并行网关 | 多个分支任务同时执行，等待全部完成后继续 | High | ✅ |
| conditional_branch | 条件分支 | 根据条件表达式选择执行路径 | Normal | ✅ |
| loop_flow | 循环 | 重复执行任务直到满足退出条件 | Normal | ✅ |
| split_join | 分合流 | 将任务拆分为多个子任务并行执行，最后聚合结果 | High | ✅ |
| compensation | 补偿事务 | 当流程失败时自动执行补偿/回滚操作 | Critical | ✅ |
| game_theory | 博弈模式 | 多智能体辩论决策模式，支持两种博弈策略 | High | ✅ |
| monitor_flow | 监控流 | 全程监控任务执行状态，实时收集性能指标 | High | ✅ |
| rule_interpreter_flow | 规则解释流 | 解析规则文档并生成执行计划 | Normal | ✅ |

## 模板详细定义

### 1. 顺序流 (seq_flow)

```yaml
template_id: seq_flow
name: 顺序流
description: 按顺序依次执行多个任务节点
qos_level: normal

execution:
  mode: sequential
  nodes:
    - task_ref: task_1
      next: task_2
    - task_ref: task_2
      next: task_3
    - task_ref: task_3
```

### 2. 并行网关 (parallel_gateway)

```yaml
template_id: parallel_gateway
name: 并行网关
description: 多个分支任务同时执行
qos_level: high

execution:
  mode: parallel
  branches:
    - branch_id: branch_a
      tasks: [task_a1, task_a2]
    - branch_id: branch_b
      tasks: [task_b1]
  join_condition: all_completed
```

### 3. 条件分支 (conditional_branch)

```yaml
template_id: conditional_branch
name: 条件分支
description: 根据条件表达式选择执行路径
qos_level: normal

execution:
  mode: conditional
  condition: "${context.value > 100}"
  branches:
    - condition: true
      tasks: [task_high]
    - condition: false
      tasks: [task_low]
```

### 4. 博弈模式 (game_theory)

```yaml
template_id: game_theory
name: 博弈模式
description: 多智能体辩论决策模式
qos_level: high

execution:
  mode: game_theory
  strategy: optimization|design
  participants:
    - agent_id: critic_agent
      role: critic
    - agent_id: creator_agent
      role: creator
    - agent_id: synthesizer_agent
      role: synthesizer
  rounds: 3
```

## v3.0 新增特性

### 插件化工作流

工作流模板支持动态加载和热更新：

```python
class WorkflowTemplateLoader:
    async def load_template(self, template_id: str) -> dict:
        """动态加载工作流模板"""
        pass
    
    async def register_template(self, template: dict) -> None:
        """注册新模板"""
        pass
```

### 分布式协调

支持跨节点工作流编排，通过分布式状态管理器保证一致性。

### 可观测性增强

每个工作流节点都有完整的调用链追踪和性能指标收集。

### 安全沙箱

工作流执行支持沙箱隔离，限制资源访问。
