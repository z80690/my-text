## 📦 Cloud Code记忆系统精髓（L3-R-MEMORY）

### 核心改进模块

| 模块 | 说明 | 优先级 |
|-----|------|-------|
| **Dream整合机制** | 定期自动整合、剪枝、更新记忆 | P0 |
| **智能检索员** | 智能匹配、相关性排序、上下文注入 | P0 |
| **记忆验证机制** | 使用前验证文件/配置存在性 | P0 |
| **记忆准入过滤** | 自动过滤可推导信息 | P0 |

### 四大核心原则

| 原则 | 说明 |
|------|------|
| **主动整理** | Dream机制：≥24小时且≥5会话自动触发整合 |
| **智能检索** | 小模型检索员，对话前注入前3-5条相关记忆 |
| **严谨验证** | "记忆说X存在"不等于"X现在存在" |
| **质量把控** | 只记录无法从代码/Git历史推导的信息 |

### 记忆类型

| 类型 | 存储路径 | 过期策略 |
|-----|---------|---------|
| User | `memories/user/` | 90天未访问标记审核 |
| Feedback | `memories/feedback/` | 180天未访问自动归档 |
| Project | `memories/project/` | 365天未访问标记审核 |
| Reference | `memories/reference/` | 无限期，手动清理 |
