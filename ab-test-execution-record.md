## 四、Bug 修复记录

### 4.1 发现的问题

| # | 问题描述 | 严重程度 | 发现阶段 | 修复方法 |
|---|---------|---------|---------|---------|
| 1 | 路由逻辑未处理带 ID 的 GET 请求 | 高 | 测试阶段 | 修改路由匹配逻辑 |
| 2 | PUT/DELETE 请求 basePath 提取错误 | 高 | 测试阶段 | 修正路径处理逻辑 |
| 3 | POST 路由参数签名不匹配 | 中 | 测试阶段 | 统一使用 (query, body) 签名 |
| 4 | 初始测试用例设计不合理 | 低 | 设计阶段 | 调整测试用例顺序 |

### 4.2 修复详情

**问题 1**: GET /api/users?id=xxx 返回 404
- **原因**: 路由逻辑只匹配 /api/users，不处理带 query string 的情况
- **修复**: 修改 GET /api/users 路由，同时处理带 ID 和不带 ID 的情况
- **验证**: ✅ 测试通过

**问题 2**: PUT /api/users?id=xxx 返回 404
- **原因**: basePath 提取使用正则 `/\/[^/]+$/` 错误地将 /users 替换为 /
- **修复**: 直接判断并添加末尾斜杠
- **验证**: ✅ 测试通过

**问题 3**: POST 路由参数签名不一致
- **原因**: 部分路由定义为 (body)，部分为 (query, body)
- **修复**: 统一为 (query, body)
- **验证**: ✅ 测试通过
